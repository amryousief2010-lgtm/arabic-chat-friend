# نظام مخازن مصنع الأعلاف الكامل

## الهدف
بناء نظام متكامل لمصنع الأعلاف يفصل بين **مخزن المواد الخام** و**مخزن العلف الجاهز للبيع**، مع شاشات لشراء الخامات وفواتير التصنيع، وداشبورد لحظي للأرصدة والأرباح. نفس النظام سيُعاد استخدامه لاحقًا في مزرعة الأمهات.

---

## 1) هيكل المخازن

داخل قسم **مصنع الأعلاف** نضيف بند **المخازن** يفتح على:

- **مخزن المواد الخام (تحت التصنيع)** — ذرة، صويا، نخالة، بريمكس، إضافات…  
  يدخل: عند شراء خامات.  
  يخرج: عند إصدار فاتورة تصنيع.
- **مخزن العلف الجاهز للبيع** — علف تسمين، علف بادي، علف بياض، بريمكس تسمين، دريس حجازي…  
  يدخل: ناتج فواتير التصنيع.  
  يخرج: عند البيع/التحويل (مرحلة لاحقة).

كل مخزن يعرض: الصنف، الرصيد (كجم + شيكارة)، متوسط التكلفة، القيمة الإجمالية، تنبيه نقص.

---

## 2) شاشة "شراء خامات"

نموذج فيه:
- المورد + رقم فاتورة المورد + التاريخ.
- بنود: الصنف (من قائمة المواد الخام) — الكمية (كجم) — سعر الكيلو — الإجمالي.
- ملاحظات + مرفق صورة الفاتورة (اختياري).
- زر "حفظ" → يضيف للمخزون + يحدّث متوسط التكلفة + يسجّل حركة شراء.

---

## 3) شاشة "فاتورة تصنيع علف"

نموذج فيه:
- المنتج الناتج (مثلاً: علف تسمين) + الكمية المستهدفة (كجم) + عدد الشكاير + وزن الشيكارة.
- بنود الخامات المستهلكة: الصنف + الكمية الفعلية.
- التكلفة تُحسب تلقائيًا من متوسط تكلفة كل خامة + تكلفة كل كجم منتج.
- زر "إنهاء التصنيع" → 
  - يخصم الخامات من مخزن المواد الخام.
  - يضيف الناتج في مخزن العلف الجاهز بمتوسط التكلفة المحسوب.
  - يسجّل دفعة تصنيع برقم batch ويحفظ السجل للطباعة.

---

## 4) داشبورد مصنع الأعلاف المحدّث

أعلى الصفحة كروت سريعة:
- إجمالي قيمة مخزن الخامات.
- إجمالي قيمة مخزن الجاهز للبيع.
- إنتاج الشهر (كجم).
- مبيعات الشهر (كجم + جنيه).
- ربح الشهر (مبيعات − تكلفة المنتج المباع).

شبكة "أرصدة الجاهز للبيع" — لكل منتج كارت يعرض: الاسم + الكمية بالكيلو + عدد الشكاير + متوسط التكلفة.

شبكة "أرصدة الخامات" — نفس الشكل.

قسم "تقرير شهري":
- جدول: الشهر / مشتريات خامات / تكلفة تصنيع / مبيعات / ربح.
- مخطط خطي للأرباح آخر 6 أشهر.

## التفاصيل التقنية

### قاعدة البيانات
- **استخدام الجداول الموجودة قدر الإمكان**:  
  - `inventory_items` (module = `feed`) — تمييز نوع المخزن بحقل جديد `warehouse_kind` ∈ (`raw_material`, `finished_feed`).  
  - `inventory_movements` للحركات (purchase / production_out / production_in / sale).  
  - `feed_production_batches` + `feed_batch_consumption` موجودة بالفعل لفواتير التصنيع.
- **جدول جديد** `feed_raw_purchases` (supplier, invoice_no, purchase_date, total, notes, created_by) + `feed_raw_purchase_items` (purchase_id, item_id, qty, unit_price, line_total).
- تريجر على إدراج بند شراء → يزيد الرصيد ويحدّث `unit_cost` كمتوسط مرجح.
- تريجر على إغلاق `feed_production_batches` (موجود غالبًا) — نتأكد أنه يخصم الخامات ويضيف الناتج بتكلفته المحسوبة.
- RLS: SELECT للأدوار أعلاه، INSERT للأدوار صاحبة الصلاحية.
- GRANT للجداول الجديدة على authenticated + service_role.

### الواجهة
- صفحة `/feed-factory/warehouses` — تبويبان: خامات / جاهز للبيع.
- صفحة `/feed-factory/purchases` — قائمة + زر "شراء جديد" يفتح Dialog النموذج.
- صفحة `/feed-factory/production-invoice/new` — نموذج فاتورة التصنيع (قد نستفيد من `FeedBatchNew` الحالية ونوسّعها).
- تحديث `FeedFactoryDashboard.tsx` بكروت القيمة + شبكتي الأرصدة + التقرير الشهري.
- بنود سايدبار جديدة تحت "مصنع الأعلاف".

### إعادة الاستخدام لمزرعة الأمهات
نفس الجداول/الشاشات ستُستنسخ بـ `module = 'farm'` لاحقًا — لذلك نبني المكوّنات قابلة لإعادة الاستخدام مع تمرير `module` كـ prop.

## نقاط أحتاج تأكيدك عليها قبل التنفيذ
1. **شيكارة كم كيلو؟** هل وزن ثابت لكل منتج (مثلاً تسمين 50 كجم، بادي 25 كجم)؟ أم نخزّن `bag_weight` لكل صنف على حدة؟
2. **المبيعات**: المصنع يبيع لمين الآن؟ (المزرعة الداخلية فقط، أم عملاء خارجيين كمان؟) هل نضيف شاشة "بيع علف" في نفس الـ scope أم نأجلها؟
3. **هل أبدأ كامل الخطة دفعة واحدة**، أم نقسّمها: أولاً (المخازن + الشراء)، ثم (فاتورة التصنيع + الداشبورد)، ثم (المبيعات والتقرير الشهري)؟

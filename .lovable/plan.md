# خطة: طلبات الكتاكيت 🐥

قسم جديد منفصل تحت "التسويق والمبيعات" لتسجيل مبيعات الكتاكيت بواسطة المنسقات الأربعة.

## 1. قاعدة البيانات

جدول جديد `chick_orders`:
- `customer_name` (اسم العميل)
- `phone_primary` (رقم الهاتف)
- `phone_secondary` (رقم آخر — اختياري)
- `governorate` (المحافظة)
- `city` (المدينة)
- `chick_age` (عمر الكتكوت بالأيام/الأسابيع)
- `chick_price` (سعر الكتكوت)
- `chick_count` (عدد الكتاكيت)
- `total_amount` (محسوب: price × count)
- `status` (pending / delivered / returned / cancelled) — افتراضي pending
- `created_by` (المنسقة المنشئة)
- `notes` (ملاحظات اختيارية)
- `created_at` / `updated_at`

## 2. صلاحيات RLS (RBAC)

| الإجراء | المنسقات (sales_moderator) | مديرة المبيعات (marketing_sales_manager / general_manager) |
|---|---|---|
| إنشاء | ✅ | ✅ |
| عرض | ✅ (طلباتها فقط) | ✅ (الكل) |
| تعديل البيانات (اسم/تليفون/سعر/عدد...) | ✅ | ✅ |
| تغيير الحالة (تسليم/مرتجع/إلغاء) | ❌ | ✅ |
| حذف | ❌ | ✅ |

تنفيذ القيد على تغيير الحالة عبر **trigger** يمنع المنسقة من تعديل عمود `status`.

## 3. الصفحات

- **`/chick-orders`** — قائمة الطلبات مع فلاتر (الحالة، التاريخ، المنسقة) + إجماليات
- **`/chick-orders/new`** — نموذج تسجيل طلب جديد
- **`/chick-orders/:id`** — تفاصيل + تعديل
- زر "تحديث الحالة" يظهر فقط لمديرة المبيعات

## 4. القائمة الجانبية

إضافة عنصر "طلبات الكتاكيت 🐥" تحت "الطلبات" في قسم التسويق والمبيعات في `SidebarMenuSections.tsx`.

## 5. التفاصيل التقنية

- نموذج بـ react-hook-form + zod للتحقق
- استخدام TanStack Query للقراءة/التحديث
- تصميم متوافق مع باقي صفحات الطلبات (RTL، ألوان البراند بنفسجي/برتقالي)
- إضافة الصفحات للراوتر في `App.tsx` مع `ProtectedRoute` للأدوار المسموحة
- ربط `created_by` تلقائياً بـ `auth.uid()` عبر default


# خطة تطوير قسم المسلخ (Slaughterhouse) وفقًا لملف Excel الفعلي

## فهم البيانات من الملف
الملف يحتوي على 3 شيتات يومية بنفس البنية المتكررة (تفريغة الذبح اليومي):

### القسم الأول: استلام النعام الحي (لكل طائر على حدة)
| # | الوزن قائم وقت الشراء | الوزن وقت الذبح | تكلفة الشراء | وقت الشراء | تكلفة العلف |
- صف لكل طائر (10–12 طائر) مع وزن وتكلفة فردية
- مثال: طائر #1 = 40.5 كجم → 5,265 ج (≈130 ج/كجم)
- إجمالي اليوم: مجموع الأوزان + إجمالي التكلفة

### القسم الثاني: تقسيمة الذبيحة (Outputs)
| م | الصنف | الكمية (كجم) | السعر | إجمالي | فرع هليوبوليس | فرع كارفور |
الأصناف المعتمدة (21 صنف) موجودة بالفعل في `slaughter_yield_standards`:
لحمة، استيك، موزة، فراشة، قطعية دبوس، دبوس بالعظم، فخدة، صندوق، تربيانكو، اسكالوب، رول النعام، فرم نعام، كبدة، قلب، رقاب، قوانص، كوارع، دهن، شغت، نخاع، ريش

### القسم الثالث: المؤشرات
- إجمالي وزن قائم vs إجمالي وزن مقطّع
- **نسبة التصافي** = الكمية المقطعة / الوزن القائم (32–38%)
- التوزيع بين الفروع (هليوبوليس / كارفور)

## الفجوات الحالية في `Slaughterhouse.tsx`

| ما هو موجود | ما هو ناقص حسب Excel |
|---|---|
| استلام إجمالي (bird_count + total_weight) | تسجيل وزن كل طائر منفصلًا |
| لا توجد تكلفة علف | حقل feed_cost لكل طائر |
| توزيع outputs على destination واحد | تقسيم نفس الصنف بين عدة فروع |
| لا يوجد ربط بـ Branches | الحاجة لجدول `branches` (هليوبوليس، كارفور، …) |
| لا يوجد تقرير يومي مطابق للورقة | تقرير "تفريغة الذبح اليومي" قابل للطباعة |
| لا توجد PDF/Excel export | تصدير التقرير اليومي |
| الربط بالمخزون/المصنع غير مؤتمت | تحويل outputs تلقائيًا لمخزون الفرع |

## التنفيذ المقترح — على دفعات

### الدفعة A — قاعدة البيانات (Migration)

1. **جدول `branches`** (مرجعي): id, code, name_ar, manager_id, address, is_active
   - بذر: هليوبوليس، كارفور (+ السماح بإضافة فروع لاحقًا)

2. **جدول `slaughter_live_birds`** (تفاصيل كل طائر داخل receipt):
   - receipt_id, bird_index, live_weight_kg, slaughter_weight_kg,
     purchase_cost, purchase_time, feed_cost, notes
   - Trigger يحسب إجمالي receipt تلقائيًا

3. **توسيع `slaughter_batch_outputs`**:
   - إضافة `branch_id` (FK → branches) — السماح بصفوف متعددة لنفس الصنف لكل فرع
   - إضافة `unit_price` (سعر بيع بدل/بجانب unit_cost)

4. **جدول `slaughter_branch_transfers`** (للربط بمخزون الفروع):
   - batch_id, branch_id, cut_name_ar, weight_kg, price, transferred_at, status
   - يتم إنشاؤه تلقائيًا عند اعتماد batch

5. **RPC: `finalize_slaughter_batch(batch_id)`**:
   - يحدّث total_meat_kg و actual_yield_pct
   - يحسب cost_per_kg_meat = (تكلفة شراء + علف) / كمية مقطعة
   - ينشئ صفوف `slaughter_branch_transfers` ويغيّر status → completed

6. **RPC: `slaughter_daily_summary(date)`** للتقرير اليومي

### الدفعة B — واجهة `Slaughterhouse.tsx`

تبويبات جديدة/محسّنة:
- **استلام الطيور**: نموذج محسّن مع جدول تفصيلي لإدخال كل طائر منفصلًا (يحاكي الـ Excel: # ، وزن، تكلفة، وقت، علف)
- **دفعات الذبح**: ربط بالاستلام، إدخال outputs مع **اختيار الفرع لكل صف**
- **التقرير اليومي**: عرض بنفس تخطيط Excel (3 أقسام) + تصدير PDF/Excel
- **التوزيع على الفروع**: تبويب يعرض transfers ويسمح بتتبع الحالة
- **مؤشرات لوحة**: نسبة التصافي اليومية/الشهرية، متوسط تكلفة الكجم، توزيع الفروع (Pie chart)

### الدفعة C — الربط بباقي النظام

| الربط | الآلية |
|---|---|
| ← المزرعة (Farm) | استلام داخلي يربط بـ `farm_transfer_id` (موجود) |
| → مصنع اللحوم (MeatFactory) | outputs تتحول لمخزون مواد خام عبر `meat_factory_raw_inventory` |
| → الفروع/المخازن (Warehouses) | `slaughter_branch_transfers` يزيد رصيد الفرع |
| → المبيعات | الأصناف المقطعة تصبح متاحة للبيع في كل فرع حسب رصيده |
| → الجودة | quality_checks مرتبطة بـ batch ✓ (موجودة) |
| → التنبيهات | تنبيه عند نسبة تصافي < 30% أو موت غير طبيعي ✓ (موجود trg_notify_low_yield) |

### الدفعة D — التقارير والتصدير
- زر "طباعة تفريغة اليوم" يولّد PDF مطابق لتنسيق Excel
- تصدير Excel بنفس البنية (3 أقسام)
- تقرير شهري: إجمالي طيور / إجمالي كجم / متوسط تصافي / إجمالي توزيع لكل فرع

## نقاط للموافقة قبل التنفيذ
1. هل لديكم بالفعل جدول فروع تحت اسم آخر؟ (لم أجد `branches` — هل أنشئه أم أستخدم `warehouses`؟)
2. هل التوزيع للفروع يكون **عند الإدخال** (المستخدم يحدد لكل صف) أم **بعد ذلك** (خطوة منفصلة)؟
3. هل تكلفة العلف تُدخل لكل طائر أم إجمالي لكل receipt؟
4. هل أبدأ بالدفعة A (Migration) فورًا؟
